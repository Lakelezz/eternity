Disclaimer:

The Key is expected to be an enum! If your key is no enum, use LimitedRequests.

Implementation

This type uses std::mem::discriminant on the key to differentiate between bucket groups.

It’s tailored to limit calling and caching HTTP endpoints, where the the key represents the endpoints containing arguments. At first, the type will convert the key to a discriminant to pick the the bucket group, ignoring the fields, and then it will use the enum with fields to find the actual bucket instance.

Warning

As said, if the key is not an enum, this type will not work as expected.
Use LimitedRequests instead for generic keys.

Inserts a single group by turning a GroupKey to a Bucket.
Returns an Err with a GroupKey if it failed converting to None.

It requires mutable ownership over self, if you know all groups before using this data structure, insert them ahead of time and use cache_or.
This approach requires immutable access and prevents needing a locking mechanism.

If you do not know all types beforehand, you will need a locking mechanism to call mut_cache_or for the mutable ownership.
It allows you to add group buckets on the run.
